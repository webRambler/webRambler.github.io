package.json 里 devDependencies和dependencies的区别
发表于2021-12-12|更新于2022-01-01|前端工具|npm
我们在使用npm install 安装模块或插件的时候，有两种命令把他们写入到 package.json 文件里面去，比如： npm i 【包名】 –save-dev npm i 【包名】 –save 在 package.json 文件里面提现出来的区别就是： 使用 npm i 【包名】 –save-dev 安装的包，会被写入到 devDependencies 对象里面去； 而使用 npm i 【包名】 –save 安装的包，则被写入到 dependencies 对象里面去。 那么 package.json 文件里面的 devDependencies 和 dependencies 对象有什么区别呢？ devDependencies 里面的插件(比如各种loader，babel全家桶及各种webpack的插件等)只用于开发环境，不用于生产环境，因此不需要打包；而 dependencies 是需要发布到生产环境的，是要打包的(比如vue，vue-router，vuex，axios等)。
